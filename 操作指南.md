# 操作指南 / User Guide

This file defines how a human must be able to use MangaMaker.

本文档定义人类用户必须能够如何使用 MangaMaker。

## 1. 启动 / Start
Web:

Web：
```bash
npm install
npm run dev
```

Desktop:

桌面：
```bash
npm run tauri dev
```

## 2. 首次进入 / First Run
The first-run flow must let a new user:

首次进入时，新用户必须能完成以下流程：

1. Create a project, or restore a local draft.
2. Create the first page.
3. See the page list, canvas, inspector, and top ribbon.
4. Understand the next step without reading code.
5. Read recommended next-step guidance from the inspector, not from a banner above the workspace.

1. 创建项目，或恢复本地草稿。
2. 创建第一页。
3. 看到页面列表、画布区、检查器和顶部 ribbon。
4. 无需阅读代码，就能理解下一步该做什么。
5. 推荐下一步提示应出现在 inspector 中，而不是占用 workspace 上方的横幅。

## 3. 页面操作 / Page Operations
The user must be able to:

用户必须能够：

1. Add a page.
2. Duplicate a page.
3. Delete a page.
4. Reorder pages.
5. Switch the current page from the left page list.

1. 添加页面。
2. 复制页面。
3. 删除页面。
4. 调整页面顺序。
5. 在左侧页面列表中切换当前页面。

## 4. 分镜操作 / Panel Operations
The user must be able to:

用户必须能够：

1. Choose the Panel tool and drag on the canvas to create a panel.
2. Select a panel, then move, resize, and delete it.
3. Turn a panel into a polygon.
4. Add or remove panel vertices.
5. Edit panel border and fill style.
6. Move panel vertices outside the comic page when the composition needs bleed or overlap.
7. **Live vertex drag preview**: When dragging a panel vertex, the panel shape must update immediately before mouse release.

1. 选择分镜工具，并在画布上拖拽创建分镜。
2. 选中分镜后移动、调整大小和删除。
3. 把分镜调整成多边形。
4. 增加或删除分镜顶点。
5. 编辑分镜边框和填充样式。
6. 在需要出血或越页构图时，把分镜顶点拖到漫画页面之外。
7. **顶点拖拽实时预览**：拖拽分镜顶点时，分镜形状必须在松开鼠标前即时更新。

Required behavior:

必需行为：

- A panel is a viewing window.
- A panel binds its own source image.
- A panel shows a cropped region from the source image, not a scaled full-image fit.
- The workspace around the page is editable, so panel geometry may extend beyond the page edge.
- **Panel Movement Rules**:
  - Left-click-drag on a panel (without image) moves the panel. After drag ends, the panel is NOT auto-selected.
  - Left-click-drag on a panel's border/edge (with image) moves the panel. After drag ends, the panel is NOT auto-selected.
  - This "drag without auto-select" behavior allows users to reposition panels without changing the current selection state.
- **Selected Panel with Image Rules**:
  - When selected, the full source image appears semi-transparent outside the panel mask.
  - Left-click-drag on the semi-transparent image pans the crop region (changes which part of image is visible).
  - The panel itself stays fixed during image pan.
  - Left-click-drag on the panel border still moves the entire panel.
- **Click vs Drag Distinction**: A "click" selects the object. A "drag" (mouse down + move + mouse up) moves the object without selecting it.
- Dragging a panel vertex or resize handle must reshape the panel without shifting the underlying image across the stage.
- When a selected panel crosses the comic-page edge, a dashed page boundary must stay visible above it so the printable area remains clear.
- Right-clicking a panel must open a custom context menu with quick panel actions such as image import/replacement, vertex editing, and delete.
- Selecting a panel must not make the workspace suddenly resize, and the selected panel must still be movable.
- 当已选中的分镜越过漫画页面边界时，页面边界虚线必须显示在分镜上方，让可导出的范围保持清晰。
- 右键点击分镜时，必须弹出自定义右键菜单，并提供导入/替换图片、顶点编辑、删除等快捷分镜操作。
- 选中分镜时，workspace 不能突然变大；并且分镜在选中后仍然必须可以移动。
- 拖拽分镜顶点或尺寸手柄时，只能改变分镜形状，不能让底下绑定图片在 stage 中跟着位移。

- 分镜的本质是一个显示窗口。
- 分镜绑定自己的源图片。
- 分镜显示的是源图裁切后的局部，而不是把整张图缩放塞进分镜。
- 页面四周的工作区也是可操作区域，因此分镜几何可以越过页面边缘。
- **分镜移动规则**：
  - 在无图片分镜上左键拖拽可移动分镜。拖拽结束后，分镜不会自动被选中。
  - 在有图片分镜的边框/边缘上左键拖拽可移动分镜。拖拽结束后，分镜不会自动被选中。
  - 这种"拖拽但不自动选中"的行为允许用户重新定位分镜而不改变当前选中状态。
- **带图分镜选中规则**：
  - 选中时，完整的源图以半透明形式显示在分镜遮罩外。
  - 在半透明图片上左键拖拽可平移裁切区域（改变分镜内可见的图片部分）。
  - 平移图片时，分镜本身保持固定。
  - 在分镜边框上左键拖拽仍然可以移动整个分镜。
- **点击与拖拽的区别**："点击"会选中对象。"拖拽"（按下+移动+释放）会移动对象但不会选中它。

## 5. 图片导入与裁切编辑 / Image Import and Crop Editing
The user must be able to:

用户必须能够：

1. Select a panel.
2. Import an image into that panel.
3. See the bound source image beyond the mask as soon as that panel is selected.
4. See the currently cut region stay highlighted while the panel remains selected.
5. Drag the image to pan the visible range.
6. Use the mouse wheel to zoom the visible range.
7. Keep the panel itself fixed in place while the image is being dragged or zoomed.
8. Unselect the panel and immediately return to normal clipped panel rendering.
- The image inside the panel must update live while the drag is still in progress.
- 在图片拖拽尚未结束时，分镜内部的可见图像就必须实时更新。
拖拽或缩放图片时，分镜本身在 stage 中的位置不能变化。

1. 选中一个分镜。
2. 把图片导入该分镜。
3. 在该分镜被选中时，立即看到遮罩之外的绑定原图内容。
4. 在该分镜保持选中时，看到当前切出区域持续高亮。
5. 通过拖拽图片来平移可见范围。
6. 通过鼠标滚轮来缩放可见范围。
7. 取消选中该分镜后，立即恢复为普通的分镜裁切显示。

## 6. 画布显示 / Canvas Display
The user must be able to see the full current page in the default full-page view without needing scroll bars.

用户必须能够在默认全页视图下，无需滚动条就看全当前页面。

Required behavior:

必需行为：

- The canvas automatically fits the visible area.
- The comic page sits inside a larger workspace instead of filling the whole canvas.
- The page should occupy roughly twenty-five percent of the visible working canvas area, leaving clear margins to the true canvas edge.
- A continuous zoom slider must allow stepless workspace scaling, with 100% representing the fit-to-workspace baseline.
- The default full-page view must not depend on scroll bars.
- When a selected text box or panel overlaps the comic-page edge, the page boundary must appear above the object as a dashed overlay.
- At higher zoom values, the workspace frame itself must still remain inside the editing surface; only the rendered content inside it should zoom.
- The area above the workspace should remain available for editing, not consumed by next-step banner text.
- Right-clicking inside the editing surface must show the product's custom context menu instead of the browser default menu.
- Crossing 100% on the zoom slider must stay visually continuous and must not suddenly resize the editing surface itself.
- 当已选中的文本框或分镜压到漫画页面边界时，页面边界必须以虚线覆盖的方式显示在对象上方。
- 缩放提高后，workspace 外框本身仍必须留在编辑区内；变化的只能是其中内容的渲染比例。
- workspace 上方区域应继续留给编辑使用，而不是被“下一步”横幅占用。
- 在编辑区域中右键点击时，必须显示产品自己的右键菜单，而不是浏览器默认右键菜单。
- 缩放滑杆跨过 100% 时，视觉变化必须连续，不能让编辑区域本身突然变大。
- 必须提供可无级调节工作区缩放的连续滑杆，其中 100% 表示适配工作区的基准比例。
- 默认全页视图不能依赖滚动条。

- 画布会自动适配可视区域。
- 漫画页面位于更大的工作区中，而不是填满整个画布。
- 页面应大致占可见工作画布面积的百分之二十五，并与真正的画布边缘保持清晰留白。
- 可以缩放，但默认全页视图不能依赖滚动条。

## 7. 文字与气泡 / Text and Bubble
The user must be able to:

用户必须能够：

1. Add a text box.
2. Edit text content.
3. Move and resize the text box.
4. Switch between horizontal and vertical text.
5. Set text horizontal alignment (left/center/right).
6. Set text vertical alignment (top/middle/bottom).
7. Add, edit, move, and resize speech bubbles.

1. 添加文本框。
2. 编辑文本内容。
3. 移动和调整文本框大小。
4. 在横排和竖排之间切换文字方向。
5. 设置文字水平对齐（左对齐/居中/右对齐）。
6. 设置文字垂直对齐（顶部/中部/底部）。
7. 添加、编辑、移动和调整气泡。
8. 切换十种气泡类型，每种都有独特的可调节属性：
   - 圆角矩形：可调节圆角半径
   - 云朵：可调节凹凸度
   - 爆炸形：可调节尖刺数量和深度
   - 锯齿形：可调节锯齿度
   - 思考气泡：可调节尾随小圆圈数量
   - 其他类型各有特色形状
9. 拖拽气泡尾巴端点，使箭头指向说话者。
10. 调整气泡边框宽度、背景颜色、边框颜色和尾巴宽度。

## 8. 字体与排版控制 / Font and Typography Controls
The user must be able to use the right sidebar Inspector to:

用户必须能够在右侧边栏 Inspector 中：

1. Choose a font family (when text or bubble is selected).
2. Change font size (when text or bubble is selected).
3. Switch between horizontal and vertical text direction (text box only).
4. Set text horizontal alignment: left, center, right.
5. Set text vertical alignment: top, middle, bottom.
6. See the locally available font list.

1. 选择字体（当选中文本框或气泡时）。
2. 调整字号（当选中文本框或气泡时）。
3. 切换横排和竖排文字方向（仅文本框）。
4. 设置文字水平对齐：左对齐、居中、右对齐。
5. 设置文字垂直对齐：顶部、中部、底部。
6. 看到本机可用字体列表。

The ribbon must also let the user change the current page background color directly.
Ribbon 还必须允许用户直接修改当前页面的背景颜色。

## 9. 导出 / Export
The user must be able to:

用户必须能够：

1. Export the current page as PNG.
2. Export the whole project as PDF.
3. Understand the last export result from the UI status area.

1. 将当前页面导出为 PNG。
2. 将整个项目导出为 PDF。
3. 从界面状态区看懂最近一次导出结果。

## 10. 语言切换 / Language Switching
The user must be able to switch the interface between Chinese and English.

用户必须能够在中文和英文之间切换界面语言。

Required behavior:

必需行为：

- Page copy, inspector copy, ribbon copy, prompts, and the governing documents must remain readable in both languages.
- Switching language must not damage project content.

- 页面文案、检查器文案、ribbon 文案、提示信息和四份根文档在中英两种语言下都必须可读。
- 切换语言不能破坏当前项目内容。

## 11. 快捷键 / Shortcuts
At minimum, the product must support:

至少必须支持：

- `V`: Select / 选择
- `P`: Panel / 分镜
- `T`: Text / 文字
- `B`: Bubble / 气泡
- `Ctrl/Cmd+Z`: Undo / 撤销
- `Shift+Ctrl/Cmd+Z`: Redo / 重做
- `Ctrl/Cmd+S`: Save / 保存

## 12. 人类验收 / Human Acceptance
Minimum human acceptance:

最低人类验收标准：

1. A new user can finish one basic comic page within minutes.
2. The user understands that a panel is a crop window, not a scale container.
3. The user can discover font, size, text direction, export, and language switching without hunting through the interface.
4. The user does not lose page context because the default view requires scrolling.
5. The user can understand that the outer margin is editable workspace, not dead space.
6. The user can understand that selecting a panel with an image reveals the source image layer for direct crop adjustment, and unselecting it restores normal panel display.
7. The user can adjust workspace zoom continuously from the ribbon instead of being limited to fixed preset percentages.
8. The user sees the panel image move live while dragging, instead of waiting for mouse release.
9. The user can reshape a selected panel without the bound image being dragged away from its stage position.
10. The user can read the dashed page boundary above overlapping selected content and understand what will export.
11. The user can change the current page background color from the ribbon without opening a separate panel.
12. The user can right-click a panel and immediately get shortcut panel actions without seeing the browser menu.
13. The user can select a panel, keep moving it, and adjust zoom past 100% without the workspace suddenly jumping larger.
14. The user still has the full workspace area available because next-step guidance no longer occupies a banner above the canvas.
10. 用户能够通过对象上方的虚线页面边界判断哪些内容会被导出，哪些不会。
11. 用户能够直接在 ribbon 中修改当前页面背景色，而不需要再打开额外面板。
12. 用户右键点击分镜时，能够直接看到快捷分镜操作，而不会再出现浏览器默认右键菜单。
13. 用户能够在选中分镜后继续移动它，并且把缩放拖过 100% 时不会看到 workspace 突然变大。
14. 用户不会再被 workspace 上方的“下一步”横幅占用画布空间，编辑区域会完整保留给排版操作。
8. 用户在拖拽图片时会立刻看到分镜内部内容实时变化，而不是等到松开鼠标。
9. 用户可以改变已选中分镜的形状，而不会把绑定图片在 stage 中一起拖偏。

1. 新用户能在几分钟内完成一页基础漫画页面。
2. 用户能理解分镜是裁切窗口，而不是缩放容器。
3. 用户能直接发现字体、字号、文字方向、导出和语言切换。
4. 默认视图不会因为必须滚动而破坏页面整体感。
5. 用户能理解页面外侧留白是可编辑工作区，而不是无效区域。
6. 用户能理解选中带图分镜时会显示源图层用于直接调整裁切，取消选中后会恢复普通分镜显示。
7. 用户能从 ribbon 中连续调节工作区缩放，而不是只能使用固定百分比预设。
